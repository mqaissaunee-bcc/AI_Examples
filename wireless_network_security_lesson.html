Why Wireless Security Matters

Wireless networks transmit data through radio waves, making them inherently more vulnerable than wired networks. Unlike wired connections where an attacker needs physical access, wireless signals can be intercepted from a distance, making security paramount.

Types of Wireless Networks

üì° WLAN (Wi-Fi)

Wireless Local Area Networks using IEEE 802.11 standards. Most common in homes and businesses.

üîµ Bluetooth

Short-range wireless technology for connecting devices (keyboards, headphones, IoT devices).

üì≤ NFC

Near Field Communication for very short-range data exchange (mobile payments, access cards).

üè∑Ô∏è RFID

Radio Frequency Identification for tracking and identification (inventory, access badges).

‚ö†Ô∏è Key Security Principle

All wireless technologies share a common vulnerability: broadcast transmission. Anyone within range can potentially intercept wireless signals, making encryption and authentication critical.

Wireless Network Attacks

üéØ Wireless LAN (WLAN) Attacks

1. Rogue Access Points

Unauthorized wireless access points installed on a network, either by malicious actors or well-meaning employees.

Scenario: An employee brings a personal wireless router and connects it to the company network to get better Wi-Fi coverage in their office. This creates an unsecured entry point into the corporate network.

2. Evil Twin Attacks

A malicious access point that mimics a legitimate one, tricking users into connecting and exposing their data.

Scenario: An attacker sets up an AP named "Starbucks_Free_WiFi" in a coffee shop. Users connect, and the attacker intercepts their credentials and browsing data.

3. Deauthentication Attacks

Forcing users to disconnect from a wireless network by sending forged deauthentication frames.

  • Can be used for denial of service
  • Forces users to reconnect, exposing handshake data
  • Prerequisite for password cracking attacks

4. Packet Sniffing

Capturing wireless traffic to analyze and extract sensitive information.

  • Captures unencrypted data in plaintext
  • Can capture WPA handshakes for offline cracking
  • Reveals network topology and connected devices

5. WPS Attacks

Exploiting Wi-Fi Protected Setup (WPS) vulnerabilities to gain unauthorized access.

  • PIN brute-force attacks (only 11,000 combinations)
  • PixieDust attack exploits weak random number generation
  • External registrar attacks

6. Jamming Attacks

Broadcasting interference on the same frequency as the wireless network to disrupt service.

üîµ Bluetooth Attacks

1. Bluejacking

Sending unsolicited messages to Bluetooth-enabled devices. Generally harmless but annoying.

2. Bluesnarfing

Unauthorized access to information from a Bluetooth device (contacts, calendar, messages).

3. Bluebugging

Taking control of a Bluetooth device to make calls, send messages, or access data.

4. BlueBorne

Air-borne attacks that spread through Bluetooth, requiring no user interaction.

üì≤ NFC & RFID Attacks

1. Eavesdropping

Intercepting NFC/RFID communications between a tag and reader.

2. Data Corruption

Interfering with NFC/RFID signals to corrupt data transmission.

3. Relay Attacks

Extending the range of NFC/RFID signals to access systems from a distance.

Scenario: An attacker uses a relay device to extend the signal from a victim's contactless payment card to make unauthorized purchases.

4. Cloning

Copying data from RFID tags to create duplicate cards or badges.

IEEE 802.11 Security Vulnerabilities

Legacy Protocol Vulnerabilities

Understanding past vulnerabilities helps us appreciate modern security measures and recognize legacy systems that may still be vulnerable.

Vulnerability Assessment

1. WEP (Wired Equivalent Privacy) - DEPRECATED

CRITICAL
  • Uses RC4 stream cipher with static keys
  • Weak 24-bit initialization vector (IV)
  • Can be cracked in minutes with tools like Aircrack-ng
  • No protection against replay attacks
  • Never use WEP in production

2. WPA (Wi-Fi Protected Access) - DEPRECATED

HIGH
  • Temporary solution to WEP's problems
  • Uses TKIP (Temporal Key Integrity Protocol)
  • Vulnerable to KRACK (Key Reinstallation Attack)
  • Susceptible to dictionary attacks with weak passwords
  • Better than WEP but should be upgraded to WPA2/WPA3

3. WPA2 (Wi-Fi Protected Access 2)

MEDIUM
  • Uses AES-CCMP for encryption (stronger than TKIP)
  • Vulnerable to KRACK attack (patchable)
  • Susceptible to offline dictionary/brute-force with weak PSK
  • No forward secrecy
  • Still widely used and acceptable with strong passwords

4. WPA3 (Latest Standard)

LOW
  • Uses SAE (Simultaneous Authentication of Equals)
  • Protection against offline dictionary attacks
  • Forward secrecy
  • 192-bit security suite for sensitive data
  • Improved encryption even on open networks

Common Misconfigurations

Default Credentials

Many routers ship with default admin passwords that users never change.

WPS Enabled

Wi-Fi Protected Setup provides convenience but introduces significant vulnerabilities.

Weak Passwords

Short or dictionary-word passwords are vulnerable to brute-force attacks.

Open Networks

Networks without any authentication allow anyone to connect.

üí° Key Concept: The 4-Way Handshake

WPA2 and WPA3 use a 4-way handshake to establish a secure connection:

  1. ANonce: AP sends a nonce (number used once)
  2. SNonce + MIC: Client responds with its nonce and message integrity code
  3. GTK: AP sends group temporal key
  4. ACK: Client acknowledges

This handshake can be captured and used for offline password cracking attacks if the PSK is weak.

Authentication Methods

PSK (Pre-Shared Key)

Also known as: WPA-Personal, WPA2-Personal

  • Single password shared by all users
  • Suitable for home and small business
  • Easier to manage but less secure for large deployments
  • Minimum 8 characters, recommend 20+

802.1X / EAP

Also known as: WPA-Enterprise, WPA2-Enterprise

  • Individual authentication for each user
  • Requires RADIUS server
  • Supports various EAP methods (PEAP, EAP-TLS, EAP-TTLS)
  • Ideal for enterprise environments

EAP Types

Common EAP Methods

  • EAP-TLS: Uses certificates for both client and server. Most secure but requires PKI infrastructure.
  • EAP-TTLS: Tunneled TLS. Only server needs certificate. Client can use password.
  • PEAP: Protected EAP. Similar to EAP-TTLS with Microsoft's implementation.
  • EAP-FAST: Flexible Authentication via Secure Tunneling. Cisco's alternative to PEAP.

üîê Encryption Algorithms

RC4: Stream cipher used in WEP and WPA-TKIP. Now considered insecure.

AES (Advanced Encryption Standard): Block cipher used in WPA2 and WPA3. Industry standard for encryption.

CCMP (Counter Mode CBC-MAC Protocol): WPA2's AES implementation.

GCMP (Galois/Counter Mode Protocol): WPA3's AES implementation. More efficient than CCMP.

Wireless Network Defense Strategies

üõ°Ô∏è Best Practices for Wireless Security

  • Use WPA3 when possible, WPA2 as minimum
  • Implement strong, unique passphrases (20+ characters)
  • Disable WPS (Wi-Fi Protected Setup)
  • Change default administrator credentials
  • Keep firmware updated on all wireless devices
  • Use separate networks for guests and IoT devices
  • Implement network segmentation (VLANs)
  • Enable MAC address filtering (as additional layer, not primary security)
  • Reduce wireless signal strength to necessary coverage area
  • Conduct regular wireless security audits

Enterprise Security Measures

802.1X Authentication

Implement enterprise authentication with RADIUS server for individual user credentials and centralized management.

Wireless IDS/IPS

Deploy wireless intrusion detection/prevention systems to identify rogue APs and suspicious activities.

Network Access Control (NAC)

Verify device compliance before allowing network access. Check for antivirus, patches, and security posture.

VPN for Sensitive Data

Require VPN connections for accessing sensitive resources, adding an extra encryption layer.

Securing Different Wireless Technologies

Bluetooth Security

  • Set devices to non-discoverable mode when not pairing
  • Use Bluetooth 4.0 or later with improved security
  • Always use pairing/PIN when available
  • Regularly update Bluetooth device firmware
  • Disable Bluetooth when not in use

NFC Security

  • Keep NFC disabled when not actively using it
  • Use shielded wallets/cases for contactless cards
  • Be aware of surroundings when making NFC transactions
  • Monitor account statements for unauthorized transactions

RFID Security

  • Implement encryption on RFID tags when possible
  • Use RFID-blocking materials for sensitive cards
  • Employ cryptographic authentication for access control
  • Regularly audit RFID tag inventory and access logs

Monitoring and Detection

What to Monitor:

  • Rogue Access Points: Unauthorized APs on your network
  • Evil Twin APs: APs impersonating legitimate ones
  • Unusual Connection Patterns: Devices connecting at odd times
  • Signal Anomalies: Jamming or interference attempts
  • Failed Authentication Attempts: Potential brute-force attacks
  • Bandwidth Usage: Unusual traffic patterns
