📋 What is IPv6?

IPv6 (Internet Protocol version 6) is the successor to IPv4, originally specified in RFC 2460 (1998) and now governed by RFC 8200 (2017). It was designed primarily to solve IPv4 address exhaustion, but it also simplifies the packet header, eliminates broadcast traffic, builds in address autoconfiguration, and restores true end-to-end connectivity without NAT.

IPv6 uses 128-bit addresses, giving 2¹²⁸ ≈ 340 undecillion (3.4×10³⁸) unique addresses — enough to assign trillions of addresses to every person on Earth. Addresses are written in hexadecimal as eight 16-bit groups (hextets) separated by colons.

2001:0db8:85a3:0000:0000:8a2e:0370:7334

Simpler than IPv4: the IPv6 header has only 8 fields versus IPv4's 13+. There is no header checksum (upper layers handle error detection), and routers never fragment packets — the source uses Path MTU Discovery instead.

🔑 Core Concepts for Network Administrators

  • Hexadecimal addressing: eight colon-separated hextets, compressible with ::.
  • /64 everywhere: virtually every LAN segment is a /64 — 64 bits network, 64 bits interface ID.
  • No broadcast: IPv6 replaces broadcast with targeted multicast groups.
  • NDP: Neighbor Discovery Protocol (ICMPv6) replaces ARP and adds router discovery.
  • SLAAC: hosts can autoconfigure addresses from router advertisements — no DHCP required.
  • Multiple addresses per interface: link-local + global + temporary privacy addresses simultaneously.
  • Link-local required: every IPv6 interface always has an fe80::/10 address.
  • Extension headers: options moved out of the base header into a flexible chain.

IPv6 Address Structure

128-bit addressing, hextets, and the two compression rules

📐 Address Anatomy

An IPv6 address is a 128-bit number written as eight hextets — 16-bit groups in hexadecimal, separated by colons. A typical global unicast address splits into three parts:

Global Routing Prefix2001:0db8:acad48 bits — assigned by ISP/RIR
+
Subnet ID000116 bits — your internal subnets
+
Interface ID0000:0000:0000:000164 bits — identifies the host

A /48 site prefix leaves 16 subnet bits = 65,536 possible /64 subnets — more subnets than most enterprises will ever use.

✂️ The Two Compression Rules

  1. Rule 1 — Drop leading zeros in each hextet: 0db8 → db8, 0000 → 0.
  2. Rule 2 — Double colon (::) replaces one contiguous run of all-zero hextets. It may appear only once per address (otherwise the expansion would be ambiguous).
2001:0db8:0000:0000:0000:0000:0000:0001
2001:db8::1  ← fully compressed (RFC 5952)

RFC 5952 canonical form: lowercase hex, leading zeros removed, :: compresses the longest zero run (the leftmost if tied), and :: is never used for a single zero hextet.

    📏 Why Hexadecimal?

    Each hex digit encodes exactly 4 bits (a nibble), so one hextet = 4 hex digits = 16 bits, and a full address = 32 hex digits = 128 bits. Writing 128 bits in dotted decimal would take 16 octets — hex keeps it (relatively) readable.

    IPv6 Address Types

    Unicast, multicast, and anycast — there are no classes and no broadcast

    Tip: The first hextet is all you need. Memorize the prefixes: 2xxx/3xxx = Global Unicast, fe80 = Link-Local, fc00/fd00 = Unique Local, ff00 = Multicast, ::1 = Loopback.

    G Global Unicast — 2000::/3

    Publicly routable, globally unique — the IPv6 equivalent of public IPv4. First three bits are 001, so addresses begin with 2 or 3. Allocated by IANA → RIRs → ISPs → customers (typically a /48 or /56 per site).

    2001:db8:acad:1::10

    L Link-Local — fe80::/10

    Every IPv6 interface has one — automatically. Valid only on the local link, never routed. Used for NDP, routing protocol adjacencies, and as the default-gateway address. The same fe80 address can exist on different links.

    fe80::1%eth0 (zone ID selects the interface)

    U Unique Local (ULA) — fc00::/7

    The rough analog of RFC 1918 private space. In practice addresses use fd00::/8 with a randomly generated 40-bit Global ID, making accidental collisions between organizations unlikely. Not routed on the public internet.

    fd12:3456:789a:1::1

    M Multicast — ff00::/8

    One-to-many delivery to a group. Replaces broadcast entirely. The second byte encodes flags and scope (1 = interface-local, 2 = link-local, 5 = site-local, e = global). See the Multicast & NDP tab.

    ff02::1 — all nodes on link

    A Anycast

    The same unicast address assigned to multiple devices; routing delivers packets to the nearest one. Indistinguishable from unicast by format. Used for DNS root servers, CDNs, and load distribution.

    S Special Addresses

    AddressPurpose
    ::1/128Loopback (the only one — vs IPv4's whole 127/8)
    ::/128Unspecified ("this host", used during DAD)

    🌳 Type Identification Decision Tree

    Look at the FIRST HEXTET:
    ├─ 2xxx or 3xxx → Global Unicast (2000::/3)
    ├─ fe80–febf → Link-Local (fe80::/10)
    ├─ fc00–fdff → Unique Local (fc00::/7)
    ├─ ffxx → Multicast (ff00::/8)
    ├─ ::1 → Loopback
    └─ :: → Unspecified

    📐 IPv6 Subnetting Is Different

    • No host math: with 2⁶⁴ interface IDs per /64, "how many usable hosts" is never the question — a /64 holds 18.4 quintillion addresses.
    • Subnet on nibble boundaries: split prefixes on 4-bit (one hex digit) boundaries — /48 → /52 → /56 → /60 → /64 — so subnets stay readable.
    • No network/broadcast reservations: the all-zeros address is the Subnet-Router anycast, but there's no broadcast address to subtract.
    • Plan by subnets, not hosts: count the /64s you need; a /48 gives 65,536 of them.

    Multicast & Neighbor Discovery

    IPv6 has no broadcast — multicast and NDP do the work instead

    🚫 No Broadcast in IPv6

    IPv4 floods every host on a segment with broadcasts (ARP, DHCP discover, etc.). IPv6 eliminates broadcast entirely: packets go only to hosts that have joined the relevant multicast group. The closest thing to a broadcast is ff02::1 (all nodes on the link), but most control traffic is far more targeted.

    🤝 Neighbor Discovery Protocol (NDP) — RFC 4861

    NDP runs over ICMPv6 and replaces ARP, ICMP router discovery, and ICMP redirect from the IPv4 world. Five message types:

    MessageICMPv6 TypePurposeIPv4 Analog
    Router Solicitation (RS)133Host asks "any routers here?" → sent to ff02::2—
    Router Advertisement (RA)134Router announces prefix, flags, default gateway → ff02::1DHCP (partially)
    Neighbor Solicitation (NS)135"Who has this IP?" → sent to solicited-node multicastARP request
    Neighbor Advertisement (NA)136"I have it — here's my MAC"ARP reply
    Redirect137Router points host to a better next hopICMP redirect

    Duplicate Address Detection (DAD): before using any address, a host sends an NS for its own address from the unspecified source ::. Silence means the address is unique and safe to use.

    Interface IDs & Autoconfiguration

    EUI-64, SLAAC, DHCPv6, and privacy addresses

    🪪 The 64-bit Interface ID

    The low 64 bits of an address identify the interface within its /64 subnet. The interface ID can come from four places:

    • Modified EUI-64 — derived from the interface's 48-bit MAC address (classic method, shown below).
    • Random / stable-private (RFC 7217) — random but stable per network; the modern OS default.
    • Temporary privacy addresses (RFC 8981) — rotate periodically for outbound connections.
    • Manual / DHCPv6 — statically assigned (e.g. ::10 for a server) or leased.

    🔧 EUI-64: MAC → Interface ID (3 steps)

    1. Split the 48-bit MAC in half: OUI (24 bits) | device (24 bits).
    2. Insert fffe in the middle to pad to 64 bits.
    3. Flip the U/L bit — the 7th bit of the first byte is inverted (XOR with 02).

    ⚡ SLAAC — Stateless Address Autoconfiguration

    1. Host generates a link-local address and verifies it with DAD.
    2. Host sends a Router Solicitation to ff02::2.
    3. Router replies with a Router Advertisement containing the /64 prefix.
    4. Host appends its interface ID to the prefix → global address. DAD again. Done — no server involved.

    RA flags steer the process: A (autonomous — use SLAAC), M (managed — get address from DHCPv6), O (other — get DNS etc. from DHCPv6).

    🆚 SLAAC vs DHCPv6

    Security note (CEH relevance): rogue RAs can hijack traffic on a LAN — mitigate with RA Guard (RFC 6105) on access ports. EUI-64 also leaks the hardware MAC, which is why privacy extensions are now default.

    Transition Mechanisms

    Running IPv6 and IPv4 side by side during the long migration

    🛤️ Three Strategies

    1. Dual Stackv4 + v6 togetherEvery device runs both protocols; applications pick per connection.
    2. Tunnelingv6 inside v4IPv6 packets encapsulated in IPv4 to cross v4-only networks.
    3. Translationv6 ↔ v4NAT64/DNS64 rewrites packets so v6-only hosts reach v4-only servers.

    Dual stack is the preferred approach wherever possible — native performance, no encapsulation overhead. "Happy Eyeballs" (RFC 8305) lets browsers race v6 and v4 connections and use whichever answers first.

    🚇 Tunneling Mechanisms

    All tunneling variants share one idea — the IPv6 packet rides as cargo inside an IPv4 packet:

    📦 IPv4 Header — Protocol = 41 · src/dst = tunnel endpoints (public IPv4)
    IPv6 Header — src/dst = the real endpoints
    Payload (TCP / UDP / ICMPv6 …)

    The IPv4 network only ever sees the outer envelope. At the far tunnel endpoint the envelope is removed and the IPv6 packet continues natively.

    🔁 NAT64 + DNS64

    For IPv6-only networks (common on mobile carriers): DNS64 synthesizes AAAA records by embedding the server's IPv4 address into the well-known prefix 64:ff9b::/96; the NAT64 gateway then translates the packets.

    📱 v6-only host2001:db8::100
    →
    🔁 NAT64 gateway64:ff9b::c000:201
    →
    🌐 v4-only server192.0.2.1

    c000:0201 is just 192.0.2.1 in hex — the IPv4 address rides inside the IPv6 address.

    📈 Deployment Reality

    • Major mobile networks in the US run IPv6-only with NAT64/464XLAT internally.
    • Google and major CDNs report roughly 40–50% of traffic over IPv6 in recent years.
    • All RIRs have exhausted their free IPv4 pools; IPv4 addresses now trade on a market.
    • Federal mandate (OMB M-21-07) requires 80% of US government IP-enabled assets to be IPv6-only by end of FY2025.

    📏 The Scale Gap

    Bar length below is proportional to the number of digits (logarithmic scale) — a linear chart is physically impossible to draw:

    IPv4
    4.3×10⁹
    IPv6
    3.4×10³⁸

    On a linear scale, if the IPv6 bar spanned this entire page, the IPv4 bar would be about 10⁻²⁷ mm wide — billions of times smaller than a proton. Put differently: IPv6 provides roughly 6.7×10¹⁷ addresses for every square millimeter of Earth's surface.

    🧠 Memory Hooks

    • 4× everything: 32→128 bits, A→AAAA records (four A's, four times the bits).
    • "Hop Limit" is honest: IPv4's TTL was never really "time" — IPv6 renamed it to what it actually counts.
    • No ARP, no broadcast, no checksum, no router fragmentation, no NAT — much of learning IPv6 is unlearning IPv4 workarounds.
