Router Fundamentals

Routers are hardware devices that send packets between different network segments. While their primary function is routing traffic, they also serve as the first line of defense in network security through packet filtering and access control.

Routing Protocols:

  • Link-state: Sends link-state advertisements (LSAs) to other routers (OSPF, IS-IS)
  • Distance-vector: Passes routing table to neighboring routers (RIP, EIGRP)
  • Path-vector: Uses dynamically updated paths between autonomous systems (BGP)

Access Control Lists (ACLs)

Access Control ListsLists of permit/deny rules that filter network traffic based on criteria such as source IP, destination IP, protocol type, and port numbers. filter traffic entering or leaving router interfaces:

Standard ACLs (1-99)
  • Filter by source IP only
  • Less granular control
  • Place close to destination
Extended ACLs (100-199)
  • Filter by source, destination, protocol, port
  • Granular traffic control
  • Place close to source
ACL Example
# Standard ACL - Deny network 173.110.0.0
Router(config)# access-list 1 deny 173.110.0.0 0.0.255.255
Router(config)# access-list 1 permit any

# Extended ACL - Allow HTTP to web server
Router(config)# access-list 101 permit tcp any host 192.168.1.100 eq 80
‚ö†Ô∏è Important ACL Rules
  • ACLs are processed top-down - order matters!
  • There is an implicit deny all at the end of every ACL
  • Wildcard mask is the inverse of a subnet mask

Understanding Firewalls

FirewallsNetwork security devices that monitor and filter incoming and outgoing network traffic based on security policies. control traffic entering and leaving a network. They can be hardware devices with embedded OS or software installed on general-purpose computers.

Hardware Firewalls
  • Faster processing
  • Higher throughput
  • Not dependent on host OS
  • Higher cost
Software Firewalls
  • Lower initial cost
  • Flexible configuration
  • Relies on OS security
  • Lower throughput
üîß

Firewall Technologies

Network Address Translation (NAT)

NATNetwork Address Translation - maps internal private IP addresses to public external addresses, hiding internal network structure. hides internal network infrastructure. PATPort Address Translation - allows thousands of internal IPs to map to one external IP using unique port numbers. extends this by using ports to track thousands of connections through one public IP.

Packet Filtering

Packet filtersFirst-generation firewalls that inspect packet headers and make allow/deny decisions based on IP addresses, ports, and protocols. screen packets based on header information: protocol type, IP addresses, and TCP/UDP ports.

Stateful Packet Inspection (SPI)

Stateful packet inspectionFirewall technology that tracks the state of network connections in a state table, determining which packets are legitimate parts of established connections. maintains a state tableA database tracking connection states including source/destination IPs, ports, sequence numbers, and connection status. that tracks connection states:

Understanding the DMZ

A Demilitarized Zone (DMZ)A network segment between internal network and Internet containing public-facing servers, providing additional protection for internal resources. is a small network containing resources available to Internet users while maintaining security on the internal network. Also called a perimeter networkAnother term for DMZ - the zone between trusted internal and untrusted external networks..

Intrusion Detection and Prevention Systems

Monitoring and protecting networks through real-time threat detection and response.

üîç

IDS vs IPS

Intrusion Detection (IDS)
  • Monitors and analyzes traffic
  • Generates alerts
  • Does NOT block traffic
  • Operates out-of-band (passive)
  • No impact on latency
Intrusion Prevention (IPS)
  • Monitors and analyzes traffic
  • Automatically blocks threats
  • Drops, resets, or redirects packets
  • Operates inline (active)
  • May add slight latency
üìç

Network-Based vs Host-Based

Network-based systemsIDS/IPS that monitor network traffic by analyzing packets across network segments using taps or span ports. monitor entire network segments by sniffing traffic.

  • Deployed at strategic network choke points
  • Sees the big picture of network traffic
  • Cannot see encrypted traffic
  • Examples: Snort, Suricata, Cisco Sourcefire

Host-based systemsIDS/IPS software installed on individual hosts monitoring system calls, logs, and file modifications. are installed directly on systems they protect.

  • Can see encrypted traffic
  • Detects local attacks
  • Resource overhead on host
  • Examples: OSSEC, Tripwire, Carbon Black
üéØ

Detection Methods

Signature-Based Detection

Compares traffic against database of known attack patterns. Similar to antivirus.

  • Pros: Low false positives, fast detection
  • Cons: Cannot detect zero-day attacks

Anomaly-Based Detection

Anomaly-based IDSDetection that establishes normal behavior baseline and alerts on significant deviations. establishes baseline of normal activity and alerts on deviations.

  • Pros: Can detect unknown (zero-day) attacks
  • Cons: Higher false positive rates

Web Filtering Technology

Attackers target user workstations with Internet access. Instead of breaking through firewalls, they trick users into visiting malicious sites. Web filteringTechnology that monitors and controls access to websites based on categories, reputation, or content analysis. addresses this threat.

How Web Filtering Works

  • URL Categorization: Sites classified by category (malware, adult, social media)
  • Reputation Scoring: Threat scores based on known malicious activity
  • Content Inspection: Real-time scanning for malicious code
  • Policy Enforcement: Allow, warn, or block based on policies

Honeypots and Deception Technology

Using decoy systems to detect, deflect, and study attacker behavior.

üçØ

What is a Honeypot?

A honeypotA decoy system designed to attract attackers, collecting information about attack techniques while diverting attention from production systems. is a computer placed on the network perimeter with information intended to lure and trap hackers. Security professionals configure these with vulnerabilities to attract attackers.

Primary Goals

  • Distraction: Divert hackers from legitimate resources
  • Detection: Identify attacks before reaching real systems
  • Intelligence: Gather info about attacker tools and techniques
  • Tracing: Keep attackers connected long enough to trace
üí° Early Warning System

Because honeypots have no legitimate traffic, any connection is inherently suspicious‚Äîmaking them excellent early warning systems.

‚öôÔ∏è

Types of Honeypots

Low-Interaction Honeypots

Emulate limited services and functionality. Easier to deploy but less detailed intelligence.

  • Simulates few vulnerable services
  • Lower risk if compromised
  • Good for detecting automated attacks
  • Examples: Honeyd, Dionaea

High-Interaction Honeypots

Real operating systems with full services that attackers can interact with.

  • Full OS and services
  • Higher risk but excellent intelligence
  • More complex to maintain
  • Example: Custom VMs, Honeynet

Virtual Honeypots

Virtual honeypotsSoftware-based honeypots that can simulate multiple systems using virtualization, making them scalable and cost-effective. are created using software rather than physical devices.

  • Can simulate entire networks
  • Cost-effective and scalable
  • Easy to deploy and reset
  • Example: Honeyd, KFSensor
