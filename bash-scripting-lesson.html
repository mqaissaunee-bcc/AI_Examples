What is Bash scripting?

Bash (Bourne Again Shell) is a command-line interpreter and scripting language for Unix and Linux systems. Bash scripts let you automate repetitive tasks, chain commands together, and build reusable workflows.

Pro tip: Start every script with a shebang line (#!/bin/bash) so the system knows which interpreter to use.

Core concepts

Common gotchas

These trip up nearly everyone learning Bash. Spot them early.

  • No spaces around = in assignments. Write name="Ada", not name = "Ada" — the spaced version runs name as a command.
  • Always quote your variables. Use "$file", not $file. Unquoted values break on spaces and get split into multiple words.
  • Spaces matter inside test brackets. [ "$x" -gt 3 ] needs spaces right after [ and before ].
  • Prefer [[ ... ]] over [ ... ] in Bash. The double-bracket form is safer with unquoted variables and supports pattern matching and &&/|| directly.
  • $(command) over backticks. Command substitution with $( ) nests cleanly and is easier to read than `command`.

Debugging your scripts

Two habits catch most bugs before they bite:

# Run a script with tracing — prints each command as it executes
bash -x myscript.sh

# Fail fast: exit on error, on unset variables, and on pipe failures
set -euo pipefail

# Check the exit status of the previous command (0 = success)
ls /tmp
echo "Exit status: $?"
Habit worth keeping: add set -euo pipefail near the top of any script that does real work. It turns silent failures into loud ones.

Practical examples

Working with variables

Variables store data you can reference and reuse throughout a script.

#!/bin/bash

# Assign variables (no spaces around =)
name="John"
age=25
city="New York"

# Display them
echo "Name: $name"
echo "Age: $age"
echo "City: $city"

# Command-line arguments
echo "First argument: $1"
echo "All arguments: $@"
echo "Number of arguments: $#"
Remember: wrap variables in double quotes to handle spaces safely and prevent word splitting.

Conditional logic

Make decisions based on conditions with if / elif / else.

#!/bin/bash

# Check whether the script runs as root
if [ "$EUID" -eq 0 ]; then
    echo "Running as root"
else
    echo "Not running as root"
fi

# String comparison
input="yes"
if [ "$input" = "yes" ]; then
    echo "User confirmed"
fi

# Numeric comparison with elif
score=82
if [ $score -ge 90 ]; then
    echo "Grade: A"
elif [ $score -ge 80 ]; then
    echo "Grade: B"
else
    echo "Grade: C or below"
fi
Test operators: -e (exists), -f (regular file), -d (directory), -r (readable), -w (writable), -x (executable).

Loops and iteration

Repeat work with for, while, and until loops.

#!/bin/bash

# For loop over a range
echo "Counting to 5:"
for i in {1..5}; do
    echo "Number: $i"
done

# For loop over a list
for color in red green blue; do
    echo "Color: $color"
done

# While loop
counter=1
while [ $counter -le 3 ]; do
    echo "Counter: $counter"
    counter=$((counter + 1))
done

# Until loop (runs until the condition becomes true)
num=3
until [ $num -eq 0 ]; do
    echo "Countdown: $num"
    num=$((num - 1))
done

Functions

Package reusable logic into named blocks that accept parameters.

#!/bin/bash

# Simple function — $1 is the first argument
greet() {
    echo "Hello, $1!"
}

# Function that "returns" a value by echoing it
add_numbers() {
    local sum=$(($1 + $2))
    echo "$sum"
}

# Call them
greet "Alice"
result=$(add_numbers 5 10)
echo "Sum: $result"
Local scope: declare variables with local inside functions so they don't leak into the global namespace.

Arrays

Store collections of values in indexed or associative arrays.

#!/bin/bash

# Indexed array
fruits=("Apple" "Orange" "Banana")

echo "First fruit: ${fruits[0]}"
echo "All fruits: ${fruits[@]}"
echo "Count: ${#fruits[@]}"

# Loop through the array
for fruit in "${fruits[@]}"; do
    echo "Fruit: $fruit"
done
Heads up: associative arrays (declare -A) need Bash 4 or newer. Index counting starts at 0.

File operations

Test file properties before acting on them.

#!/bin/bash

filename="test.txt"

if [ -e "$filename" ]; then
    echo "File exists"
    if [ -f "$filename" ]; then
        echo "It's a regular file"
    fi
    [ -r "$filename" ] && echo "Readable"
    [ -w "$filename" ] && echo "Writable"
else
    echo "File does not exist"
fi

# Read a file line by line
while IFS= read -r line; do
    echo "Line: $line"
done < "$filename"
