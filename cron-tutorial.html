üí° Pro Tip

Each asterisk (*) represents "every" value in that field. Combine them with operators like comma (,), dash (-), and slash (/) for powerful scheduling!

üî£ Cron Operators

*
All potential values in a field
,
List multiple values (e.g., 1,3,5)
-
A range of values (e.g., 1-5)
/
Specify a step value (e.g., */5 = every 5 units)
L
Last value (only in month and day-of-week)
W
Closest weekday from given time
#
Only in day-of-week (e.g., 5#3 = 3rd Friday)
?
No specific value (day-of-month/day-of-week)

‚è∞ The AT Command

What is AT?

The at command is used to schedule a one-time task to run at a specific time in the future. Unlike cron (which runs recurring jobs), at executes a command once and then forgets about it.

at 10:00 PM
Schedule a job for 10:00 PM today
at now + 5 minutes
Schedule a job 5 minutes from now
at 2:00 AM tomorrow
Schedule a job for tomorrow at 2 AM
at 14:00 12/25/2025
Schedule for specific date and time
at noon next week
Schedule for noon next week

Using AT - Interactive Example

$ at 10:30 PM
at> /home/user/backup.sh
at> <Ctrl+D>
job 3 at Sat Nov 2 22:30:00 2025

üîß ATD Daemon & Commands

The ATD Daemon

The atd daemon is the background service that executes jobs scheduled with the at command. It must be running for at jobs to execute.

$ sudo systemctl start atd
Start the atd daemon
$ sudo systemctl enable atd
Enable atd to start on boot
$ sudo systemctl status atd
Check atd daemon status
$ atq
List all pending at jobs
$ at -l
Alternative: list pending jobs
$ atrm <job_number>
Remove/cancel a scheduled job
$ at -c <job_number>
Display the contents of a job
$ at -f script.sh now + 1 hour
Schedule a script file to run

üí° Time Specifications

Absolute: 10:00 AM, 14:30, midnight, noon, teatime (4 PM)

Relative: now + 2 hours, now + 30 minutes, tomorrow

Special: today, tomorrow, next week, next month

‚öñÔ∏è Cron vs AT: Quick Comparison

Feature Cron AT
Purpose Recurring scheduled tasks One-time future tasks
Frequency Repeating (daily, weekly, etc.) Runs once and exits
Daemon crond atd
Configuration Crontab files (persistent) Command line (temporary)
Time Syntax Five-field format (* * * * *) Natural language (now + 2 hours)
Best For Regular maintenance, backups, reports Reminders, delayed tasks, one-off jobs
Persistence Survives reboots Jobs persist until executed
Example 0 2 * * * backup.sh at 2am <<< backup.sh

üéØ When to Use What?

Use CRON when:

  • Tasks need to run regularly (daily, weekly, monthly)
  • You need consistent scheduling (same time every day)
  • Jobs should persist across system reboots
  • You're automating system maintenance

Use AT when:

  • You need to run a task once in the future
  • You want to schedule something ad-hoc
  • You need a reminder or delayed action
  • Scheduling is more convenient in natural language
