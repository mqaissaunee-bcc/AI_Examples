How This Tool Works
A workflow for turning chapter coverage into a verified, traditional exam.

  
🔄 Workflow

    
    
      
  1. Configure — set the exam header and add one slot per chapter/topic. Choose the problem type, points, parts, and figure mode for each.
    2. 
      
  2. Generate & run the prompt — copy the prompt into your LLM (Claude recommended for LaTeX and figure specs). It returns one structured problem per slot.
    3. 
      
  3. Review & verify — validate the JSON, then check each problem: confirm the computation check passes, the figure matches the given values, and the worked solution is correct. Mark each verified.
    4. 
      
  4. Export — download the student exam, answer key, and Canvas page.
    5. 
    

  

  
🧮 Figures

    
Each problem can carry one figure, chosen per problem in the review step:

    
    
      
  • Mechanism (auto-drawn) — four-bar linkage or slider-crank. Enter link lengths and the input angle; the tool solves the position and draws it. The same values must appear in the problem, so the picture and answer key stay locked. If the links can't physically close, it shows a notice instead of a wrong figure.
    • 
      
  • SVG / AI (any figure, on the fly) — describe a figure (“cantilever beam, 8 kN tip load, label the moment”) and generate it with AI, then hand-edit the SVG with a live preview; or paste your own. The LLM that writes the problem can also emit the figure directly. Markup is sanitized and baked into exports as static SVG. Uses your own Anthropic API key, stored only in your browser.
    • 
      
  • Image upload — embed any diagram (free-body, gear train, 3D, photo) directly in the file as a self-contained image.
    • 
      
  • Exhibit block — a monospace panel for a router config, code snippet, packet table, or text topology. Best for networking and cybersecurity problems.
    • 
      
  • None / caption — no figure, or a caption describing one you'll provide separately.
    •
