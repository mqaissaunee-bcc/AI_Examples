How This Tool Works
A workflow for turning chapter coverage into a verified, traditional exam.

  
🔄 Workflow

    
    
      
  1. Configure — set the exam header and add one slot per chapter/topic. Choose the problem type, points, parts, and figure mode for each.
    2. 
      
  2. Generate & run the prompt — copy the prompt into your LLM (Claude recommended for LaTeX and figure specs). It returns one structured problem per slot.
    3. 
      
  3. Review & verify — validate the JSON, then check each problem: confirm the computation check passes, the figure matches the given values, and the worked solution is correct. Mark each verified.
    4. 
      
  4. Export — download the student exam, answer key, and Canvas page.
    5. 
    

  

  
🧮 Figures

    
Each problem can carry one figure. There are three modes, chosen per problem in Review:

    
    
      
  • Figure (image) — the workhorse. The LLM writes a precise image-generation prompt for the figure; you copy it (one at a time in Review, or all at once from the Figures tab), generate the picture in your image tool of choice, then upload it. The image embeds into the student exam, answer key, and Canvas page. The Figures tab also takes a bulk drop of images and assigns them to the prompts in filename order.
    • 
      
  • Exhibit block — a monospace panel for a router config, code snippet, packet table, or text topology. Rendered as text, not an image. Best for networking and cybersecurity problems.
    • 
      
  • None — no figure.
    • 
    

    
A shared style line on the Figures tab is appended to every prompt so your figures look consistent. Image models render text and exact numbers unreliably, so prompts keep labels short and leave the given values in the problem text — always check each generated figure against the problem before exporting.
