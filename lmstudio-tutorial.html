Key Terms Explained
Plain-language glossary for complete beginners.
7B / 13B / 70B
The number of "parameters" — like the brain's connections. More = smarter, but needs more RAM and runs slower.
Parameter
A single number in the model that was "learned" during training. A 7B model has 7 billion of them.
Quantization
Compressing the model to save space — like JPEG for images. Q4 = smaller/faster, Q8 = bigger/better quality.
GGUF
The file format LM Studio uses. Think of it like ".mp3 for AI models" — it's the universal container.
MLX
Apple's own AI framework. MLX models run much faster on M1/M2/M3/M4 Macs than standard formats.
Context Length
How much text the model can "remember" at once. 4K ≈ a few pages. 128K ≈ an entire book.
Temperature
How creative / random the AI is. 0.1 = very precise and factual. 1.0+ = creative and unpredictable.
VRAM
Memory on your graphics card (GPU). Loading model layers here is much faster than regular RAM.
RAG
Retrieval-Augmented Generation — lets the AI read your documents and answer questions about them.

Beyond First Chat
What to explore after you're up and running.

A system prompt is a hidden instruction you give the AI before any conversation starts. It shapes the AI's personality, tone, and focus.

💡In LM Studio, click the ⚙️ System Prompt tab above the chat window. Type your instructions there.

Example system prompts:

"You are a patient tutor. Explain everything simply, use analogies, and never assume prior knowledge."
"You are a coding assistant. Always provide working code examples. Explain what each part does."
"You are a creative writing partner. Be imaginative, suggest unexpected plot twists, and match my writing style."

LM Studio can run as a local API server — meaning other apps on your computer can use your AI model, just like they'd call ChatGPT's API. It's free and stays private.

  1. In LM Studio, click the ↔️ Local Server tab in the left sidebar.
  2. Select your loaded model from the dropdown at the top.
  3. Click Start Server. The server starts on http://localhost:1234.
  4. Apps that support the OpenAI API format (like Open WebUI, Cursor, or custom scripts) can now connect to your local model.
✅This is how you connect LM Studio to tools like VS Code extensions, Obsidian AI plugins, or Python scripts for advanced workflows.

RAG (Document Q&A) lets you drag a PDF, Word doc, or text file into the chat — and then ask questions about it. The AI reads your document and answers based on it.

  1. First, download the Nomic Embed Text v1.5 embedding model from the Discover tab. This is required — it's what "reads" your documents.
  2. In LM Studio settings, under AI Chat, set your embedding model to Nomic Embed Text.
  3. In the Chat tab, click the 📎 Attach button and drag in your document (PDF, .txt, .md, .docx).
  4. Ask questions about the document: "Summarize this document" or "What does it say about X?"

Once you find settings you like, save them as a preset so you don't have to reconfigure every time you open LM Studio.

  1. Configure your model settings (temperature, context length, system prompt, etc.) to your liking.
  2. In the model settings panel, look for the 💾 Save Preset or Presets dropdown.
  3. Give your preset a name (e.g., "Creative Writing" or "Code Helper") and save it.
  4. Next time you open LM Studio, load your preset from the dropdown — all settings restore instantly.
💡Great for educators: create one preset per assignment type and share the settings with students.

LM Studio is the most beginner-friendly option, but you may encounter these alternatives in tutorials and communities.

Ollama
Command-line tool for running models. No GUI — you type commands in a terminal. More flexible for developers building their own apps; less beginner-friendly for day-to-day use. If you see tutorial instructions like ollama run llama3, that's Ollama.
Jan
A desktop app similar to LM Studio with a cleaner chat UI. Also supports cloud models alongside local ones. Worth trying if you want a simpler chat-only interface without the developer features.
GPT4All
Desktop app focused on very low hardware requirements. Good for older machines or if LM Studio feels too heavy. Fewer models and settings, but works on hardware that others won't.
💡You don't need to choose one forever — many people use LM Studio for exploration and Ollama for scripting. They can be installed side by side.

Common Mistakes & Fixes
Things that trip up most beginners.
❌Model runs very slowly: GPU layers may not be enabled. In LM Studio → model settings, set n_gpu_layers to -1. Verify your GPU is recognized under Settings → Hardware.
❌"Out of memory" crash: Your model is too large for your RAM. Switch to a smaller quantization (e.g., Q8 → Q4_K_M) or pick a smaller model (e.g., 13B → 7B).
❌Responses cut off mid-sentence: Increase Max Tokens in generation settings. The default may be too low for long responses.
⚠️Model repetition / loops: Increase Repeat Penalty to 1.15 or higher.
⚠️Slow download: Models are large (2–8 GB). Use a wired connection if possible. Downloads resume automatically if interrupted.
⚠️LM Studio UI looks different from the guide: LM Studio updates regularly. If something looks unfamiliar, check the LM Studio Discord or docs at lmstudio.ai/docs.
💡Apple Silicon tip: Select the MLX or Metal-optimized version when available. These are dramatically faster on M1/M2/M3/M4 chips.
✅Best practice: Start with a Q4_K_M quantized model — best balance of quality, speed, and RAM for most computers.

Vision & Multimodal Models
Models that can read images, not just text.

Some models can accept an image as input alongside text. You can upload a photo and ask "What is in this image?" or "Describe this chart." These are called vision-capable or multimodal models.

LLaVA 1.6 7B
Classic open-source vision model. Can describe images, read text in photos, and answer visual questions. Search: llava-v1.6-mistral-7b. Needs 8 GB RAM.
Qwen2-VL 7B
Alibaba's strong vision model. Excellent at reading charts, tables, documents, and handwriting. Search: qwen2-vl-7b-instruct. Needs 8 GB RAM.
Llama 3.2 11B Vision
Meta's multimodal Llama. Strong reasoning about images with long context. Search: llama-3.2-11b-vision. Needs 12–16 GB RAM.
Moondream 2
Tiny 1.8B vision model — very fast, surprisingly capable for basic image description. Works on 4 GB RAM. Search: moondream2.
💡In LM Studio, load a vision model then click the 📎 image attach button in the chat window to upload your image alongside your text prompt.

Responsible Use
Especially important for educational settings.

Running AI locally removes platform-level content filters. This is powerful — and comes with responsibility, particularly when working with students.

⚠️No content moderation: Local models have no built-in filters. The same model that helps students write essays can produce harmful content if prompted to. Think carefully about student access to local AI tools without supervision.
⚠️Hallucinations: Local models can confidently state false information. They are tools for thinking, not authoritative sources. Always verify factual claims — especially in academic or clinical contexts.
💡For classroom use: Consider using system prompts to constrain behavior: "You are a tutor. Do not do homework for students. Guide them with questions instead." This won't guarantee safety but sets useful defaults.
✅Privacy advantage: Unlike cloud AI, local models never send student data to third-party servers — a real benefit for FERPA compliance and institutional data policy.

Understanding Quantization
What do Q4, Q8, GGUF, MLX mean?

Quantization compresses AI models by reducing the precision of each number — like JPEG vs RAW for images. Smaller = faster to run, but slight quality loss.

FormatFile SizeQualityBest For
Q2_KTiny⭐⭐Very low RAM systems (4–6 GB). Noticeable quality drop.
Q4_K_MSmall⭐⭐⭐⭐✅ Sweet spot. Best balance of quality and size. Start here.
Q5_K_MMedium⭐⭐⭐⭐½16 GB+ RAM, slightly better quality.
Q8_0Large⭐⭐⭐⭐⭐Near-perfect quality. Needs 16–32 GB RAM.
GGUFVariesVariesUniversal format for all platforms (llama.cpp based).
MLXVaries⭐⭐⭐⭐⭐Apple Silicon only — fastest option on Mac M-series.
