The Origin Story

SCORM Shareable Content Object Reference Model ‚Äî a set of technical standards that allow e-learning content to work across different learning management systems. stands for Shareable Content Object Reference Model. It was developed by the ADL Initiative Advanced Distributed Learning Initiative ‚Äî a research and development program sponsored by the U.S. Department of Defense to standardize and modernize training delivery. (Advanced Distributed Learning), a program sponsored by the United States Department of Defense, starting in 1999.

The military needed a way to share training content across different systems and vendors. Before SCORM, e-learning content created for one system couldn't work in another‚Äîa costly problem when you're training millions of personnel across hundreds of platforms.

Version History

  • SCORM 1.0 (2000) ‚Äî Initial release, proof of concept
  • SCORM 1.1 (2001) ‚Äî First practical implementation
  • SCORM 1.2 (2001) ‚Äî The "sweet spot" version, still the most widely supported today
  • SCORM 2004 (2004) ‚Äî Added sequencing and navigation, but more complex
Why We Use SCORM 1.2 SCORM 1.2 offers the best balance of functionality and compatibility. It's supported by virtually every LMS (Canvas, Blackboard, Brightspace, Moodle, etc.) and handles the essentials: launching content, tracking completion, and recording scores.

What SCORM Actually Does

SCORM provides three key capabilities:

  1. Packaging ‚Äî A standard way to bundle your content (HTML, images, etc.) with a manifest file that tells the LMS what's inside
  2. Communication ‚Äî A JavaScript API Application Programming Interface ‚Äî a set of functions that allow your content to "talk" to the LMS, reporting scores and completion status. that lets your content talk to the LMS (report scores, track progress)
  3. Sequencing ‚Äî Rules for how learners navigate through content (primarily SCORM 2004)

The Key Files in a SCORM Package

A SCORM package is simply a ZIP file containing:

  • imsmanifest.xml ‚Äî The "table of contents" that describes your content
  • index.html ‚Äî Your actual learning content
  • Any supporting files (CSS, JavaScript, images)

Quick Retrofit (Manual Method)

If you prefer to add SCORM support yourself, you only need to add two lines of code at the point where your app calculates the final score:

üíª Add These Two Lines
// Add these right after calculating the final score: window.QUIZ_SCORE = score; // Store score (0-100) window.QUIZ_COMPLETE = true; // Signal completion

Then use the SCORM Packager tool (or the conversion prompt) to wrap your app‚Äîthe packager will automatically detect these variables.

Importing SCORM Packages into Your LMS

Once you have your SCORM ZIP file, follow the instructions for your LMS Learning Management System ‚Äî the platform where you host and deliver courses (e.g., Canvas, Blackboard, Brightspace). below.

Canvas

Primary LMS

Canvas has excellent SCORM support through its built-in import feature.

Import Steps

  1. Open Course Settings
    Navigate to your course and click Settings in the left sidebar
  2. Go to Import
    Click Import Course Content on the right side of the Settings page
  3. Select Content Type
    From the "Content Type" dropdown, select SCORM Package
  4. Upload Your File
    Click Choose File and select your SCORM ZIP file
  5. Start Import
    Click Import and wait for processing (usually under a minute)
  6. Add to Module
    Go to Modules, click + to add an item, select External Tool, and choose your imported SCORM content
Canvas Tips
  • SCORM scores automatically sync to the Canvas gradebook
  • Set point value when adding to a module for grade weight
  • Students can re-attempt by having you reset their submission
  • View detailed SCORM data in the SpeedGrader

Brightspace (D2L)

Full Support

Import Steps

  1. Navigate to Content
    Go to your course and click Content in the navbar
  2. Add a Module
    Create or select a module where you want the SCORM content
  3. Upload Package
    Click Upload/Create ‚Üí Upload Files
    Select your SCORM ZIP file and upload
  4. Alternative: Import Tool
    Or use Course Admin ‚Üí Import/Export/Copy Components ‚Üí Import Components ‚Üí from a file
  5. Configure Activity
    Click on the imported content to configure grade settings and attempts
Brightspace Tips
  • Create a grade item first if you want automatic grade sync
  • Set completion tracking in the activity settings
  • Use the "Class Progress" tool to monitor SCORM completion

Blackboard

Google Classroom

Limited Support
Important: Google Classroom does not natively support SCORM
Google Classroom is not a full LMS Learning Management System ‚Äî a platform designed to host, deliver, and track educational content and assessments. and lacks built-in SCORM support. However, there are workarounds.

Option 1: Host Elsewhere + Link

The simplest approach‚Äîhost your SCORM content on an external platform and link to it from Google Classroom.

  1. Upload to a SCORM Host
    Use a service like SCORM Cloud (cloud.scorm.com), Amazon S3, or your institution's web server
  2. Get the Launch URL
    Most SCORM hosting services provide a shareable link
  3. Add to Classroom
    In Google Classroom, create an Assignment and add the link
Limitation Scores will not automatically sync to Google Classroom's gradebook. Students would need to report their scores, or you'd check them in the SCORM hosting platform.

Option 2: Use an LTI Integration

Some third-party tools provide LTI Learning Tools Interoperability ‚Äî a standard that allows external tools to integrate with learning platforms, enabling features like single sign-on and grade passback. bridges that work with Google Classroom:

  • SCORM Cloud ‚Äî Offers LTI integration (paid)
  • iSpring ‚Äî Converts and hosts SCORM with sharing options
  • Articulate Rise/Storyline ‚Äî Publish with link sharing

Option 3: Convert to Non-SCORM Format

For simple quizzes, consider rebuilding in Google Forms, which integrates natively with Classroom and the gradebook.

Recommendation If you heavily use Google Classroom and need proper SCORM tracking, consider advocating for a full LMS (Canvas, Schoology, etc.) or using SCORM Cloud as a bridge.

SCORM Quick Reference

Key terms and concepts at a glance.

Essential SCORM Terms

Term Definition
SCO Shareable Content Object ‚Äî a single learning unit that can communicate with the LMS
Manifest The imsmanifest.xml file that describes package contents
API The JavaScript interface for LMS communication
cmi.core.score.raw Data element storing the learner's score (0-100)
lesson_status Completion state: incomplete, completed, passed, or failed
mastery_score The minimum score required to "pass" (set in manifest)

The SCORM API Functions

These are the JavaScript functions your content uses to communicate with the LMS:

SCORM 1.2 API Reference
// Initialize communication with LMS API.LMSInitialize(""); // Set a value API.LMSSetValue("cmi.core.lesson_status", "incomplete"); API.LMSSetValue("cmi.core.score.raw", "85"); // Get a value var status = API.LMSGetValue("cmi.core.lesson_status"); // Save data to LMS API.LMSCommit(""); // End session API.LMSFinish("");

Lesson Status Values

  • not attempted ‚Äî Learner hasn't started
  • incomplete ‚Äî Learner started but hasn't finished
  • completed ‚Äî Learner finished (no score judgment)
  • passed ‚Äî Learner finished with score ‚â• mastery score
  • failed ‚Äî Learner finished with score < mastery score

Minimum Viable imsmanifest.xml

This is the simplest manifest file structure for a single-SCO package:

imsmanifest.xml Template
<?xml version="1.0" encoding="UTF-8"?> <manifest identifier="YOUR_UNIQUE_ID" version="1.0" xmlns="http://www.imsproject.org/xsd/imscp_rootv1p1p2" xmlns:adlcp="http://www.adlnet.org/xsd/adlcp_rootv1p2"> <metadata> <schema>ADL SCORM</schema> <schemaversion>1.2</schemaversion> </metadata> <organizations default="org1"> <organization identifier="org1"> <title>Your Content Title</title> <item identifier="item1" identifierref="resource1"> <title>Your Content Title</title> <adlcp:masteryscore>70</adlcp:masteryscore> </item> </organization> </organizations> <resources> <resource identifier="resource1" type="webcontent" adlcp:scormtype="sco" href="index.html"> <file href="index.html"/> </resource> </resources> </manifest>

Troubleshooting Common Issues

Score not showing in gradebook

  • Ensure LMSSetValue("cmi.core.score.raw", score) is called
  • Call LMSCommit("") after setting values
  • Make sure score is a number between 0-100
  • Check that the LMS grade column is linked to the SCORM item

Content shows "Not Attempted" even after completion

  • Verify lesson_status is being set to "completed", "passed", or "failed"
  • Ensure LMSFinish("") is called when leaving the page
  • Check browser console for JavaScript errors

Package won't import

  • Confirm the ZIP contains imsmanifest.xml at the root level
  • Check that the manifest XML is well-formed (no syntax errors)
  • Ensure file references in manifest match actual file names

Content displays but doesn't communicate

  • Verify the API wrapper is finding the LMS API object
  • Check that LMSInitialize("") is called on load
  • Some LMS platforms require specific popup settings‚Äîtry disabling popup blockers
