Anatomy of a
Claude Prompt

Every well-crafted Claude prompt is built from layered components. Understanding each layer helps you write prompts that are clear, targeted, and effective.

claude.ai · new conversation
① System Prompt
You are an expert curriculum designer helping community college faculty develop inclusive, engaging course materials aligned to learning outcomes.
② Persona / Role
Act as a supportive instructional coach who is direct, practical, and avoids academic jargon whenever possible.
③ Context
I'm redesigning my Intro to Psychology syllabus for fall 2026. Students are mostly first-generation, working adults in evening classes. Class meets twice a week for 75 minutes.
④ Task / Instruction
Draft three active-learning activities for the first unit on Research Methods. Each should take 10–15 minutes and require no special materials.
⑤ Examples (Few-Shot)
Good: "Think-Pair-Share: students write a hypothesis, discuss with a partner, share with class." Bad: "Watch a 20-minute documentary."
⑥ Output Format
Return each activity as: Name · Time · Materials · Step-by-step instructions · Why it works (1 sentence).
⑦ Constraints / Guardrails
Do not suggest activities requiring smartphones, internet, or prior readings. Keep language at an 8th-grade reading level.
① System Prompt
Sets Claude's overall role and context. Placed before conversation begins. Invisible to end-users in deployed apps.
② Persona / Role
Shapes Claude's voice, tone, and expertise level. "Act as…" framing calibrates formality, depth, and style.
③ Context
Background info Claude needs to answer well. Who, what, where, when — the more specific, the better the output.
④ Task / Instruction
The core ask. Use action verbs: draft, summarize, compare, list. One clear task avoids ambiguous outputs.
⑤ Examples (Few-Shot)
Show Claude what "good" looks like. Even one example dramatically improves quality. Showing beats describing.
⑥ Output Format
Specify structure: bullet list, table, JSON, markdown. Prevents Claude from guessing what form you need.
⑦ Constraints
What to avoid, length limits, audience rules, tone. Negative instructions trim the output space just as well.
🧱
Not all layers are required
A simple question might only need a task. Complex tasks benefit from all seven. Start minimal and add layers when output quality suffers.
🔁
Order matters — mostly
Claude reads top-to-bottom. Put system/role context early and your actual task near the end where it's freshest in attention.
🎯
Iterate, don't re-write
If output misses the mark, diagnose which layer is weak — context? format? task? Fix that layer specifically and re-run.
