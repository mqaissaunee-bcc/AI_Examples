LLM Guide for Quantitative Questions

    
Quantitative question generation requires careful LLM selection and human verification of all calculations. Here's what you need to know.

  


  

    
⚠ Calculation Accuracy Warning

    
All current LLMs make arithmetic and algebraic errors. No LLM can be trusted to produce 100% numerically correct quantitative questions without human verification. Common failure modes across STEM, Business, Health Sciences, and Technology:

    
    
      
  • Arithmetic errors in multi-step calculations (especially with fractions, exponents, trig)
    • 
      
  • Inconsistent units — mixing SI and US, or dropping units mid-calculation
    • 
      
  • Incorrect distractor derivation — distractors that don't correspond to real student errors
    • 
      
  • LaTeX rendering errors — malformed expressions that don't render correctly
    • 
      
  • Self-contradiction — stem assumes one scenario, worked solution solves a different one
    •

  1. Generate questions → paste JSON into Step 3 → validate
    2. 
      
  2. For each numerical_mc question: expand the card (it auto-opens) and click ▼ Hide Worked Solution to verify every step. Check the formula used, the substituted values, the arithmetic, significant figures, and units. When satisfied, click the ⚠ Needs Verification badge — it turns green when marked verified.
    3. 
      
  3. Use 🔍 Find Teaching Resources to locate authoritative explanations and worked examples for the concept being tested
    4. 
      
  4. Expand the worked solution toggle on each card — read through every step
    5. 
      
  5. Check that each distractor corresponds to a real student error (documented in distractor_logic)
    6. 
      
  6. Verify LaTeX renders correctly in the preview (KaTeX is shown above each question)
    7. 
      
  7. Edit any question with errors using the ✎ button
    8. 
      
  8. Export only after verifying all questions
