Installation Media

Enterprise Linux distributions provide multiple forms of installation media to accommodate different deployment scenarios. Understanding these options allows administrators to select the most appropriate method for their environment.

Types of Installation Media

The binary ISO 9660 format image file contains the complete Anaconda installation program along with the BaseOS and AppStream package repositories.

  • Contains all packages needed for a complete installation without network access
  • Typical size: 8-10 GB
  • Ideal for environments with limited or no network connectivity
  • Can be burned to physical DVD or mounted as virtual media
‚ÑπÔ∏è
Note

The binary DVD contains both the BaseOS repository (core operating system packages) and the AppStream repository (applications, development tools, and language runtimes).

A smaller boot ISO image contains only the Anaconda installer and requires network access to package repositories available via HTTP, FTP, or NFS.

  • Typical size: 600-900 MB
  • Requires configured network during installation
  • Always installs the latest available packages from the repository
  • Ideal for environments with reliable network connectivity

A QCOW2 image provides a prebuilt system disk ready for immediate deployment as a virtual machine.

  • No installation required - ready to boot immediately
  • Standard format for KVM-based virtualization
  • Ideal for cloud and enterprise virtual environments
  • Supports thin provisioning and snapshots

System Requirements

Disk Space Requirements

A minimum of 10 GiB of disk space is required on the partition where the operating system will be installed. Additional space may be required depending on:

  • Selected software packages and environment groups
  • User data and application requirements
  • Log file growth and temporary storage needs

Memory Requirements

The minimum RAM requirement varies based on the installation method:

Recommended Minimum RAM by Installation Type
Installation Type x86-64, ARMv8, z14 POWER9
Local media (USB, DVD) 1.5 GiB 3 GiB
NFS network installation 1.5 GiB 3 GiB
HTTP, HTTPS, or FTP installation 3 GiB 4 GiB
üí°
Best Practice

For production servers, allocate significantly more RAM than the minimum requirements. Consider the workload requirements, installed services, and expected growth when sizing memory.

Troubleshooting the Installation

During installation, Anaconda provides virtual consoles for monitoring and troubleshooting. The tmux terminal multiplexer provides five windows within the first virtual console.

Installation Console Access Key Sequences
Key Sequence Function
Ctrl+Alt+F1 Access the tmux terminal multiplexer
Ctrl+B then 1 Main installation information page
Ctrl+B then 2 Root shell prompt (logs in /tmp directory)
Ctrl+B then 3 Display /tmp/anaconda.log contents
Ctrl+B then 4 Display /tmp/storage.log contents
Ctrl+B then 5 Display /tmp/program.log contents
Ctrl+Alt+F6 Return to Anaconda graphical interface

Introduction to Kickstart

The Kickstart feature automates system installations by providing a text file containing all configuration directives. This approach eliminates manual intervention during deployment and ensures consistent configurations across multiple systems.

Key Benefits

  • Enables unattended, hands-off installations
  • Ensures consistent configuration across all deployed systems
  • Reduces deployment time and human error
  • Supports customization through pre and post-installation scripts
  • Comparable to Windows unattended installation answer files

Kickstart File Structure

Command Section

Installation commands that define how to install the system (required commands like language, keyboard, timezone)

%packages Section

Specifies which software packages, groups, and modules to install or exclude

%pre Section

Scripts that execute before disk partitioning (optional)

%post Section

Scripts that execute after installation completes (optional)

‚ö†Ô∏è
Important

Comment lines begin with the hash character (#). Additional sections start with a percentage sign (%). Each section ends with the %end directive. Missing required values will cause the installer to prompt interactively or abort entirely.

Essential Kickstart Commands

Complete Kickstart Example

#version=RHEL9 # System bootloader configuration bootloader --append="console=ttyS0" --location=mbr --boot-drive=vda # Clear and partition disks clearpart --all --initlabel ignoredisk --only-use=vda zerombr autopart # Installation source and method text repo --name="BaseOS" --baseurl="http://content.example.com/rhel9/BaseOS/" repo --name="AppStream" --baseurl="http://content.example.com/rhel9/AppStream/" url --url="http://content.example.com/rhel9/dvd/" # Localization settings keyboard --vckeymap=us lang en_US # Security and authentication rootpw --plaintext redhat selinux --enforcing firstboot --disable # Time configuration timezone America/New_York --utc # Package selection %packages @core chrony firewalld httpd -plymouth %end # Post-installation script %post --erroronfail echo "Kickstarted on $(date)" >> /etc/issue systemctl enable httpd %end

Package Specification Syntax

Package Specification Syntax
Prefix Meaning Example
(none) Individual package by name httpd
@ Package group (by name or ID) @web-server
@^ Environment group @^minimal-environment
- Exclude package or group -plymouth

KVM Virtualization

Virtualization enables running multiple operating systems on a single physical machine. Enterprise Linux supports KVM (Kernel-based Virtual Machine), a full virtualization solution integrated directly into the Linux kernel.

KVM Architecture Overview

Virtual Machines

Guest OS instances (Linux, Windows) running independently

KVM Hypervisor

Kernel module providing virtualization infrastructure

Physical Hardware

CPU, Memory, Storage, Network interfaces

Installing Virtualization Tools

# Install the virtualization host group [root@host ~]# dnf group install "Virtualization Host" # Verify system requirements [root@host ~]# virt-host-validate # Install web console VM plugin [root@host ~]# dnf install cockpit-machines

Creating Virtual Machines

# Create a VM with virt-install [root@host ~]# virt-install --name demo \ --memory 4096 \ --vcpus 2 \ --disk size=40 \ --os-variant rhel9.0 \ --cdrom /root/rhel.iso

Lab 1: Create a Basic Kickstart File

Beginner

Objectives

  • Create a minimal Kickstart configuration file
  • Validate the Kickstart file syntax
  • Understand required vs optional directives
1 Create the Kickstart file ‚ñº

Create a new file called minimal.ks with basic configuration:

[student@workstation ~]$ cat > minimal.ks <<'EOF'
# Minimal Kickstart File
lang en_US
keyboard --vckeymap=us
timezone America/New_York --utc
rootpw --plaintext testpassword
bootloader --location=mbr
clearpart --all --initlabel
autopart
%packages
@core
%end
EOF
‚úì Verification

Verify the file was created:

[student@workstation ~]$ ls -l minimal.ks
-rw-r--r--. 1 student student 256 Nov 25 10:00 minimal.ks
2 Validate the Kickstart file ‚ñº

Use the ksvalidator command to check for syntax errors:

[student@workstation ~]$ ksvalidator minimal.ks
# No output indicates the file is valid
üí°
Tip

If ksvalidator is not installed, install it with: dnf install pykickstart

3 Add network configuration ‚ñº

Enhance your Kickstart file with network settings:

[student@workstation ~]$ cat >> minimal.ks <<'EOF'
network --bootproto=dhcp --device=eth0 --onboot=yes
network --hostname=server.example.com
firewall --enabled --service=ssh
EOF

Lab 2: Create a Virtual Machine

Intermediate

Objectives

  • Verify virtualization support on the host
  • Create a new virtual machine using virt-install
  • Manage the VM using virsh commands
1 Verify virtualization support ‚ñº

Run the validation command to check if the system supports KVM:

[root@host ~]# virt-host-validate
QEMU: Checking for hardware virtualization : PASS
QEMU: Checking if device /dev/kvm exists : PASS
QEMU: Checking if device /dev/kvm is accessible : PASS
2 Create a virtual machine ‚ñº

Use virt-install to create a new VM (replace ISO path as needed):

[root@host ~]# virt-install --name testvm \
  --memory 2048 \
  --vcpus 2 \
  --disk size=20 \
  --os-variant rhel9.0 \
  --location /path/to/rhel9.iso \
  --extra-args "inst.ks=file:/minimal.ks"
3 Manage the VM with virsh ‚ñº

Practice common VM management commands:

[root@host ~]# virsh list --all
Id Name State
1 testvm running

[root@host ~]# virsh shutdown testvm
[root@host ~]# virsh start testvm
[root@host ~]# virsh console testvm

Complete Glossary

  • Anaconda
    The graphical installation program used by Enterprise Linux distributions for configuring and installing the operating system.
    Installation
  • BaseOS Repository
    The core repository containing essential operating system packages required for basic system functionality.
    Installation
  • AppStream Repository
    Repository containing applications, development tools, and language runtimes that extend BaseOS functionality.
    Installation
  • Kickstart
    An automated installation method using a configuration file that provides answers to all installation prompts for unattended deployment.
    Kickstart
  • ksvalidator
    A command-line utility that validates Kickstart file syntax, checking for errors before deployment.
    Kickstart
  • %post Section
    A Kickstart section containing scripts that execute after installation completes, used for system customization.
    Kickstart
  • KVM
    Kernel-based Virtual Machine - a full virtualization solution built into the Linux kernel for running multiple guest operating systems.
    Virtualization
  • QCOW2
    QEMU Copy-On-Write version 2 - a disk image format supporting thin provisioning, snapshots, and compression.
    Virtualization
  • virt-install
    A command-line tool for creating new virtual machines with specified CPU, memory, storage, and network configurations.
    Virtualization
  • virsh
    The primary command-line interface for managing virtual machines, including starting, stopping, and configuring VMs.
    Virtualization
