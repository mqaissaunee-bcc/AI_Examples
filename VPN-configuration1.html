Configuring a Site-to-Site VPN

A Hands-On Simulation for Network Security

🎯 Learning Objectives

By the end of this interactive lesson, you will be able to:

  • Explain the purpose of ISAKMP (Phase 1) and IPsec (Phase 2) in establishing a secure tunnel.
  • Identify the core Cisco IOS commands required to configure a basic site-to-site VPN.
  • Configure a virtual router's cryptographic policies using a simulated Command Line Interface (CLI).

1. The Real-World Scenario

Imagine you are a network administrator for a growing medical supply company. You have your headquarters in New York, and you just opened a new branch office in Los Angeles. Employees in LA need to access patient databases securely over the public internet.

To achieve this, you need to set up a VPN. Specifically, a Site-to-Site IPsec VPN.

A Site-to-Site VPN acts like a secure, invisible pipe connecting the two networks. To build this pipe, routers must agree on how to secure the data. This negotiation happens in two distinct phases.

2. Understanding Phase 1 & Phase 2

ISAKMP (Phase 1): The Secure Handshake

Before we can send secure data, the two routers must establish a secure channel just to talk to each other. This is called the ISAKMP phase. They agree on:

  • Encryption: How will we scramble the data? (e.g., AES)
  • Hash: How do we know data hasn't been tampered with? (e.g., SHA)
  • Authentication: How do we prove who we are? (e.g., Pre-Shared Key)

IPsec (Phase 2): The Data Tunnel

Once Phase 1 is secure, the routers use it to negotiate Phase 2: the actual IPsec tunnel where user data (like database queries) will flow. Here, you define a Transform Set, which dictates the encryption for the payload.

4. Interactive Configuration Lab

It's time to configure Router A (New York). You will access the router via SSH and enter the correct Cisco IOS commands to establish Phase 1 of the VPN.

Lab Tasks

Follow these steps to configure the ISAKMP Policy. Type the commands exactly as shown into the terminal.

  1. Enter privileged mode:
    enable
  2. Enter global configuration mode:
    configure terminal
  3. Create an ISAKMP policy with sequence 10:
    crypto isakmp policy 10
  4. Set encryption to AES:
    encryption aes
  5. Set authentication to pre-shared key:
    authentication pre-share
  6. Exit back to global config:
    exit

5. Deep Dive: Transform Sets

If you want to understand how Phase 2 is configured, the Transform Set is crucial. It defines the specific protocols used to protect the data payload.

6. Troubleshooting your VPN

Sometimes, the tunnel just won't come up. The most common commands used to troubleshoot an IPsec VPN on a Cisco device are:

  • show crypto isakmp sa - Shows if Phase 1 successfully established.
  • show crypto ipsec sa - Shows if Phase 2 is active and if packets are being encrypted/decrypted.
